ID: SEC-003
Title: XSS via dangerouslySetInnerHTML in ComposerPromptEditor
Severity: Medium
CVSS: CVSS:3.1/AV:N/AC:L/PR:N/UI:R/S:C/C:L/I:L/A:N (5.4)
Category: A03:2021 Injection / CWE-79
Status: Confirmed

Location:
- file: apps/web/src/components/ComposerPromptEditor.tsx
- function: ComposerInlineChip
- lines: 265

Affected entry points:
- GET /api/chat -> apps/web/src/components/ComposerPromptEditor.tsx:265 -> React render (line 265)

Description:
The application uses `dangerouslySetInnerHTML` to render an SVG icon string (`SKILL_CHIP_ICON_SVG`). If this SVG string is ever made dynamic or influenced by an attacker, it could include `<script>` tags or `onload` handlers that execute arbitrary JavaScript.

Vulnerable code:
```tsx
<span
  aria-hidden="true"
  className={COMPOSER_INLINE_CHIP_ICON_CLASS_NAME}
  dangerouslySetInnerHTML={{ __html: SKILL_CHIP_ICON_SVG }}
/>
```

Attack scenario:
1. Attacker finds a way to modify the constant or input that defines `SKILL_CHIP_ICON_SVG`.
2. The UI renders the malicious SVG.
3. The embedded script executes.

Proof of concept:
Modify `SKILL_CHIP_ICON_SVG` to include a script payload:
```javascript
const SKILL_CHIP_ICON_SVG = '<svg onload=alert(1)></svg>';
```
When rendered, the javascript is executed.

Impact:
Cross-Site Scripting (XSS).

Root cause:
Unsafe usage of `dangerouslySetInnerHTML`.

Remediation:
Instead of `dangerouslySetInnerHTML`, render the SVG directly as React components or use a safe icon library.

References:
