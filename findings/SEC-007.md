ID: SEC-007
Title: SQLite In-Memory Database Usage in Production
Severity: Informational
CVSS: N/A
Category: Misconfiguration
Status: Confirmed

Location:
- file: apps/server/src/persistence/NodeSqliteClient.ts
- function: makeMemory
- lines: 191

Affected entry points:
- Database initialization -> apps/server/src/persistence/NodeSqliteClient.ts:191 -> SQLite Native Driver

Description:
The node SQLite client defines a `makeMemory` client which connects to `:memory:`. While useful for testing, if this is accidentally or maliciously configured in a production environment, all data will be lost upon application restart.

Vulnerable code:
```typescript
const makeMemory = (
  config: SqliteMemoryClientConfig = {},
): Effect.Effect<Client.SqlClient, never, Scope.Scope | Reactivity.Reactivity> =>
  makeWithDatabase(
    {
      ...config,
      filename: ":memory:",
      readonly: false,
    },
```

Attack scenario:
This is more of an operational risk than a direct attack, unless an attacker can manipulate the configuration to force the application to use an in-memory database, thereby causing a denial of service (data loss).

Proof of concept:
N/A - Configurable setting.

Impact:
Data loss on restart.

Root cause:
Availability of an in-memory database configuration without strict environment guards.

Remediation:
Ensure that `makeMemory` is strictly guarded and only used in non-production environments (e.g., via environment variable checks).

References:
- CWE-16: Configuration
