ID: SEC-006
Title: Server Secret Store Race Condition / Predictability
Severity: Medium
CVSS: CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:N/A:N (5.5)
Category: A02:2021 Cryptographic Failures / CWE-330
Status: Confirmed

Location:
- file: apps/server/src/auth/Layers/ServerSecretStore.ts
- function: set
- lines: 46

Affected entry points:
- Server secret generation -> apps/server/src/auth/Layers/ServerSecretStore.ts:46 -> FileSystem.writeFile (line 48)

Description:
The server secret store uses `Crypto.randomUUID()` to generate a temporary file name: `const tempPath = \`${secretPath}.${Crypto.randomUUID()}.tmp\`;`. UUIDv4 is not cryptographically secure. While the file permissions are subsequently set to `0o600`, an attacker with local file system access (or an arbitrary file write primitive) could potentially guess the temporary file name and attempt to read it or overwrite it before the rename occurs, leading to a race condition or information disclosure.

Vulnerable code:
```typescript
const tempPath = `${secretPath}.${Crypto.randomUUID()}.tmp`;
```

Attack scenario:
1. Attacker predicts the UUID generated for the temporary file.
2. Attacker continuously attempts to read from or write to the predicted file path.
3. If successful, the attacker can either steal the secret or overwrite it with a known value.

Proof of concept:
An attacker monitors the file system and attempts to read the `.tmp` file:
```bash
while true; do cat .secrets/*.tmp; done
```

Impact:
Information disclosure or tampering with server secrets.

Root cause:
Use of predictable randomness for temporary file generation in a sensitive context.

Remediation:
Use `crypto.randomBytes(16).toString('hex')` to generate a cryptographically secure random suffix for the temporary file.
