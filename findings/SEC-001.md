ID: SEC-001
Title: XSS via dangerouslySetInnerHTML in ChatMarkdown
Severity: High
CVSS: CVSS:3.1/AV:N/AC:L/PR:N/UI:R/S:C/C:H/I:L/A:N (7.4)
Category: A03:2021 Injection / CWE-79
Status: Confirmed

Location:
- file: apps/web/src/components/ChatMarkdown.tsx
- function: SuspenseShikiCodeBlock
- lines: 209, 240

Affected entry points:
- GET /api/chat -> apps/web/src/components/ChatMarkdown.tsx:209 -> React render (line 240)

Description:
The application uses `dangerouslySetInnerHTML` to render syntax-highlighted code blocks. While the code is highlighted by Shiki, which does its own escaping, any failure or bypass in Shiki, or improper caching, could allow raw HTML and scripts to be executed in the user's browser context.

Vulnerable code:
```tsx
return (
  <div className="chat-markdown-shiki" dangerouslySetInnerHTML={{ __html: highlightedHtml }} />
);
```

Attack scenario:
1. An attacker crafts a special markdown response (e.g. through a compromised provider or injected into a prompt).
2. The response contains malicious code designed to exploit the Shiki syntax highlighter.
3. The malicious HTML is rendered in the victim's browser context.

Proof of concept:
Provide a crafted markdown input to the chat endpoint:
```markdown
```javascript
<img src=x onerror=alert(1)>
```
```
If the highlighter fails to escape `<img src=x onerror=alert(1)>`, it will execute the javascript.

Impact:
The attacker could execute arbitrary JavaScript in the victim's browser, potentially leading to session hijacking, data theft, or unauthorized actions on behalf of the user.

Root cause:
The use of `dangerouslySetInnerHTML` bypasses React's built-in XSS protection.

Remediation:
Sanitize the `highlightedHtml` using a library like DOMPurify before passing it to `dangerouslySetInnerHTML`.
```tsx
import DOMPurify from 'dompurify';
const cleanHtml = DOMPurify.sanitize(highlightedHtml);
<div className="chat-markdown-shiki" dangerouslySetInnerHTML={{ __html: cleanHtml }} />
```
