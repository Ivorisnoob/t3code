ID: SEC-002
Title: Command Injection Vector on Windows via openInEditor
Severity: Critical
CVSS: CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H (9.8)
Category: A03:2021 Injection / CWE-78
Status: Confirmed

Location:
- file: apps/server/src/open.ts
- function: launchDetached
- lines: 196-202

Affected entry points:
- POST /api/openInEditor -> apps/server/src/open.ts:196 -> launchDetached -> child_process.spawn (line 200)

Description:
On Windows platforms, the `spawn` command is executed with `shell: true`. The arguments passed to `spawn` are manually quoted: `launch.args.map((a) => '"' + a + '"')`. This is insufficient to prevent command injection in `cmd.exe`. If a malicious `cwd` or filename contains a double quote (`"`), it can break out of the quote block and inject arbitrary shell commands (e.g. `"malicious" & calc.exe & "path"`).

Vulnerable code:
```typescript
const isWin32 = process.platform === "win32";
child = spawn(
  launch.command,
  isWin32 ? launch.args.map((a) => `"${a}"`) : [...launch.args],
  {
    detached: true,
    stdio: "ignore",
    shell: isWin32,
  },
);
```

Attack scenario:
1. An attacker manages to create a directory or file with a malicious name containing `"` and command separators like `&` or `|`.
2. The attacker triggers an action that causes the application to open that file/directory in an editor (e.g., clicking an attachment link).
3. The `cwd` or target path is passed to `openInEditor`.
4. The `launch.args` are quoted with `""`, but the payload breaks out: `"C:\malicious" & calc.exe"`.
5. The command is executed by `cmd.exe`, spawning the payload.

Proof of concept:
Invoke the API to open a file with a crafted name:
```json
{
  "cwd": "C:\temp\file" & calc.exe & "",
  "editor": "vscode"
}
```

Impact:
Remote Code Execution (RCE) on the victim's machine with the privileges of the application.

Root cause:
Inadequate escaping of command-line arguments when using `shell: true` on Windows.

Remediation:
Avoid `shell: true` if possible. If required, properly escape all special characters for `cmd.exe` or use a well-tested escaping library. Since `spawn` on Windows without `shell: true` can execute `.cmd` and `.bat` files in recent Node versions, try removing `shell: true` entirely.

```typescript
child = spawn(
  launch.command,
  [...launch.args], // Don't manually quote
  {
    detached: true,
    stdio: "ignore",
    shell: false, // Turn off shell execution
  },
);
```
